# 用户管理API接口文档
## 一、文档说明

本文档基于**MongoDB数据库**的用户表结构设计，包含用户注册、用户登录、修改密码三个核心接口，遵循RESTful规范，接口请求/响应数据格式均为`JSON`，适用于村民管理系统的用户模块开发。

### 数据库表（users）核心字段
| 字段名    | 类型     | 说明                   | 约束         |
| --------- | -------- | ---------------------- | ------------ |
| _id       | ObjectId | 主键ID                 | 系统自动生成 |
| loginName | 字符串   | 登录账号（唯一）       | 非空         |
| loginPwd  | 字符串   | 登录密码               | 非空         |
| realName  | 字符串   | 真实姓名               | 可空         |
| phone     | 字符串   | 手机号                 | 可空         |
| address   | 字符串   | 家庭住址               | 可空         |
| role      | 字符串   | 用户角色（默认：村民） | 非空         |
| state     | 数字     | 账号状态 0-正常 1-禁用 | 非空         |

---

## 二、通用规范
1. **请求域名**：`http://xxx.xxx.xxx:端口`（开发/测试环境自行替换）
2. **请求头**：`Content-Type: application/json`
3. **响应格式**
```json
{
  "code": 200,        // 状态码：200成功 400参数错误 401未授权 500服务器异常
  "msg": "操作成功",  // 提示信息
  "data": {}          // 业务数据（无数据时为null）
}
```

---

## 三、接口详情
### 1. 用户注册接口
- **接口地址**：`/api/user/register`
- **请求方式**：`POST`
- **接口描述**：新用户注册，校验登录账号唯一性，默认角色为村民，账号状态正常
- **请求参数**

| 参数名    | 类型   | 是否必填 | 说明             |
| --------- | ------ | -------- | ---------------- |
| loginName | 字符串 | 是       | 登录账号（唯一） |
| loginPwd  | 字符串 | 是       | 登录密码         |
| realName  | 字符串 | 否       | 真实姓名         |
| phone     | 字符串 | 否       | 手机号           |
| address   | 字符串 | 否       | 家庭住址         |

- **请求示例**
```json
{
  "loginName": "liuyang",
  "loginPwd": "123456",
  "realName": "张二狗",
  "phone": "13388996652",
  "address": "太阳村二队"
}
```

- **响应示例（成功）**
```json
{
  "code": 200,
  "msg": "注册成功",
  "data": {
    "_id": "69c23a6fd8589fba7cb98a23",
    "loginName": "liuyang",
    "realName": "张二狗",
    "role": "村民",
    "state": 0
  }
}
```

- **响应示例（失败）**
```json
{
  "code": 400,
  "msg": "登录账号已存在",
  "data": null
}
```

---

### 2. 用户登录接口
- **接口地址**：`/api/user/login`
- **请求方式**：`POST`
- **接口描述**：用户账号密码登录，校验账号状态，返回用户信息
- **请求参数**

| 参数名    | 类型   | 是否必填 | 说明     |
| --------- | ------ | -------- | -------- |
| loginName | 字符串 | 是       | 登录账号 |
| loginPwd  | 字符串 | 是       | 登录密码 |

- **请求示例**
```json
{
  "loginName": "liuyang",
  "loginPwd": "123456"
}
```

- **响应示例（成功）**
```json
{
  "code": 200,
  "msg": "登录成功",
  "data": {
    "_id": "69c23a6fd8589fba7cb98a23",
    "loginName": "liuyang",
    "realName": "张二狗",
    "phone": "13388996652",
    "address": "太阳村二队",
    "role": "村民",
    "state": 0
  }
}
```

- **响应示例（失败）**
```json
{
  "code": 401,
  "msg": "账号或密码错误",
  "data": null
}
```
```json
{
  "code": 401,
  "msg": "账号已被禁用，请联系管理员",
  "data": null
}
```

---

### 3. 修改密码接口
- **接口地址**：`/api/user/updatePassword`
- **请求方式**：`POST`
- **接口描述**：已登录用户修改密码，校验原密码正确性
- **请求参数**

| 参数名      | 类型   | 是否必填 | 说明     |
| ----------- | ------ | -------- | -------- |
| loginName   | 字符串 | 是       | 登录账号 |
| oldPassword | 字符串 | 是       | 原密码   |
| newPassword | 字符串 | 是       | 新密码   |

- **请求示例**
```json
{
  "loginName": "liuyang",
  "oldPassword": "123456",
  "newPassword": "654321"
}
```

- **响应示例（成功）**
```json
{
  "code": 200,
  "msg": "密码修改成功，请重新登录",
  "data": null
}
```

- **响应示例（失败）**
```json
{
  "code": 400,
  "msg": "原密码错误",
  "data": null
}
```
```json
{
  "code": 400,
  "msg": "用户不存在",
  "data": null
}
```

---

## 四、状态码说明
| 状态码 | 含义                      |
| ------ | ------------------------- |
| 200    | 操作成功                  |
| 400    | 请求参数错误/业务校验失败 |
| 401    | 未授权/登录失效/账号禁用  |
| 500    | 服务器内部异常            |
